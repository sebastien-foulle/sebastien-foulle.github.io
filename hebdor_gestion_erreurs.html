Introduction

Dans des simulations longues, par exemple le calcul de modeles predictifs en parcourant une grille d’hyperparametres, on a envie de pouvoir

  • intercepter les warnings et erreurs pour eviter l’interruption de la simulation
  • stocker les messages associes
  • conserver les resultats calcules dans le cas de simples warnings

Gestion des erreurs et warnings avec du R basique

La solution proposee quand on execute la commande demo("error.catching", ask = FALSE) est totalement incompre peu intuitive.

tryCatch.W.E <- function(expr) {
  W <- NULL
  # warning handler
  w_handler <- function(ww){ 
    W <<- ww
    invokeRestart("muffleWarning")
  }
  list(value = withCallingHandlers(tryCatch(expr, error = function(e) e),
                                   warning = w_handler),
       warning = W)
}
tryCatch.W.E(log(2))
$value
[1] 0.6931472

$warning
NULL
tryCatch.W.E(log(-1))
$value
[1] NaN

$warning
<simpleWarning in log(-1): production de NaN>
tryCatch.W.E(log("a"))
$value
<simpleError in log("a"): argument non numérique pour une fonction mathématique>

$warning
NULL

La vignette https://cran.r-project.org/web/packages/tryCatchLog/vignettes/tryCatchLog-intro.html du package tryCatchLog permet de comprendre le fonctionnement des fonctions withCallingHandlers, tryCatch et invokeRestart utilisees ici.

Gestion des erreurs et warnings avec le package catchr

La gestion devient beaucoup plus simple : on intercepte les warnings et erreurs avec muffle, ce qui evite les messages rouges dans la console et les plantages en cas d’erreur. Et avec collect on stocke les messages generes.

library("catchr")
collect_warnings_errors <- make_catch_fn(warning = c(collect, muffle),
                                         error = c(collect, muffle))
collect_warnings_errors(log(2))
$value
[1] 0.6931472

$warning
list()

$error
list()
collect_warnings_errors(log(-1))
$value
[1] NaN

$warning
$warning[[1]]
<simpleWarning in log(-1): production de NaN>


$error
list()
collect_warnings_errors(log("a"))
$value
